# Ever-Pretty 钉钉智能助手 · 使用说明书

**版本：** 2026-06-12（面向业务同事，无需任何技术背景）

---

## 一、这是什么

在钉钉群里 **@机器人，用一句平常话提问**，它会自动去公司数据库查数、分析，几分钟内把带表格的分析结果回到群里。

> 例：`@EverPretty 智能助手 EP-US 近 14 天退货原因 Top 10`
>
> 几分钟后机器人回复一张「现状 / 主因分析 / 建议」三段式分析卡片。

不需要写 SQL、不需要找数据组排期 —— 直接问就行。

## 二、有哪几个机器人，该问谁

公司一共有 **7 个机器人**，各管一摊，分别驻在对应的钉钉群里。**问对机器人，答案更快更准。**

|---|---|---|
| **EverPretty 智能助手**（总台） | 综合问题、跨部门决策、日常查数 | 「EG02084 该不该停售」「公司这个月 GMV 多少」「EP-US 退货原因 Top 10」 |
| **Finance Bot** | 利润、成本、ROI、定价 | 「EE02968 利润空间多少」「这款的广告投入产出划算吗」 |

| **Supply Bot** | 补货、库存、停售、周转 | 「哪些款该优先补货」「EP-US 有哪些断货风险」 |
| **CX Ops Bot** | listing、客户反馈、客服、复购 | 「客户对 EG02778 主要抱怨什么」「listing 描述和实际反馈一致吗」 |

## 三、怎么提问

### 基本动作

在群里输入 `@机器人名 + 你的问题`，发送即可。中文英文都可以。

### 提问三要素（不写也行，有默认值）

| 要素 | 怎么写 | 不写时的默认 |
|---|---|---|
| **店铺** | EP-US / EP-UK / PZ-US / DAMA-US（品牌-国家） | EP-US |
| **时间** | 近 7 天 / 上周 / 本月 / 5月1日到5月15日 | 近 14 天 |
| **数量** | Top 5 / 前 20 | Top 10 |

机器人会在回复里注明它用了哪些默认值（如「默认按 EP-US / 近 14 天」），不符合预期直接追问改条件即可。

### 提问小技巧

- **款号**写 7 位编码（如 `EG02778`、`EE02559`）；**ASIN** 写 B0 开头 10 位编码，两种都认。
- **问单天的数据要说清楚**：「6月1号当天的销量」，而不是只说「6月1号」。
- **一次问一件事**结果最好；复合决策类问题（该不该停售这种）交给总台。
- 想看公司整体数字直接问：「公司这个月销售额多少」「各平台 GMV 对比」。

## 四、发出问题后会发生什么

1. **几秒内**：机器人回「🤔 思考中...」，确认收到。
2. **过程中**：会有「📝 进度」卡片同步它正在查什么数据，不用催。
3. **完成后**：以**一条新消息**推送最终答案（不是回复在原消息下面），通常带数据表格，末尾注明数据来源。

**耗时预期：**

| 问题类型 | 一般耗时 |
|---|---|
| 快捷指令（见第六节） | 几秒 |
| 普通查数 / 分析 | **1–5 分钟** |
| 跨部门决策（总台派单给多个专家） | 5–10 分钟 |

⚠️ 机器人是**排队处理**的：同一个机器人同时只处理一个问题，前面有人在问时你的问题会排在后面。高峰期请耐心等，**不要重复发同一个问题**（会重复排队，更慢）。

如果超过 15 分钟没结果，机器人会主动发一条超时提示（带一个 issue 编号）。把这个编号发给管理员即可追查。

## 五、追问和多轮对话

机器人**记得你和它的对话上下文**（按人、按群独立，别人的提问不会串进你的对话）：

- 上一轮问了「EP-US Top 20 畅销」，这一轮直接说「**近 7 天呢**」「**换 EP-UK**」即可，不用重复完整问题。
- 想换一个全新话题、或感觉机器人被之前的话题带偏了，发送 **`重置`**（或 `清空` / `reset` / `clear`），即可清空你的对话记忆，重新开始。

## 六、快捷指令（秒回，不用等分析）

以下固定格式的指令走快速通道，几秒返回原始数据：

| 指令 | 返回 |
|---|---|
| `查 ASIN B0XXXXXXXX` | 该 ASIN 的销量/评分/评论等主数据 |
| `查 SKU EE00001-US12 from 2026-04-01` | 该 SKU 自某日起的订单聚合 |
| `最近订单` / `近 7 天订单` | Amazon 最新订单流 |
| `EP-UK Top 5 畅销` | 该店近 14 天 GMV 排名前 N 的 SKU |
| `EP-US 库存预警` | 高速动销、有断货风险的 SKU |
| `查账户 138486201` | Meta 广告账户余额/状态 |
| `帮助` | 指令列表 |

## 七、对机器人的建议进行表态（闭环）

机器人给出的建议会带 `S1`、`S2` 这样的编号。你可以直接在群里回复表态，系统会记录归档：

- **采纳**：`采纳 S1 S3`（或 `接受 S1`）
- **拒绝**：`拒绝 S2`（或 `否决 S2`）

## 八、数据口径须知（读懂答案必看）

| 须知 | 说明 |
|---|---|
| **数据新鲜度** | Amazon 数据是 **T+1**（看昨天及更早）；独立站销量当天可见。 |
| **金额单位** | 销售额 / GMV 一律 **USD**（仓库已统一折算），不分币种。 |
| **款退货率** | 按「销售月成熟口径」计算：一个月卖出的货要等约 45 天退货充分回流后才算数。所以**新款 / 最近一两个月会回答「暂无可信退货率」—— 这是正常的**，不是机器人坏了，硬给数字反而是错的。 |
| **两种"退款率"不要混** | 「某款退货率」= 件数口径（退货件/销售件）；「公司/平台退款率」= 订单口径（有退款的订单占比）。机器人会按你的问法自动选，但对比数字时注意口径。 |
| **独立站（官网）** | 只有**销量件数**，没有金额；独立站的**退货率暂未接入**，问了会直说没有。 |
| **「有销量」≠「在线」** | 销量是订单事实，不代表 listing 此刻一定在售。 |
| **数据刷新窗口** | 北京时间 **9:00 / 13:00 / 17:30** 前后数据库在全量刷新，机器人可能回「数据源刷新中，请稍后重试」—— 等几分钟再问即可。 |

## 九、它做不到的事

- ❌ **任何写操作**：改价、上下架、改库存、改广告出价 —— 它只读数据，不动数据。
- ❌ 独立站退货率 / 退货原因（数据源未接入）。
- ❌ 实时物流在途状态。
- ❌ 跨账号合并视图（每次查询锁定单店，可以分两问）。
- ❌ 实时查美/法/德/澳官网商品目录（实时商品查询目前仅 UK 站）。

## 十、常见问题 FAQ

**Q：@了机器人没反应？**
A：等 1 分钟（可能在排队）。仍没有「思考中」回执，说明机器人离线了，联系管理员。

**Q：回了「思考中」但很久没答案？**
A：复杂问题最长 10 分钟左右；超过 15 分钟机器人会自己发超时提示。期间不要重复提问。

**Q：答案看起来不对？**
A：先核对口径（第八节），尤其是时间范围和店铺默认值；仍有疑问，把机器人的回复截图 + 你的原问题发给管理员核查。

**Q：机器人答非所问 / 一直绕在旧话题上？**
A：发 `重置` 清空对话，再重新提问，并把条件（店铺/时间/款号）写完整。

**Q：能私聊机器人吗？**
A：可以在群里问任何与该群主题相关的问题；不确定的问题统一发给总台「EverPretty 智能助手」。

**Q：我的提问别人看得到吗？**
A：在群里 @ 提问，群成员都能看到问题和答案；对话记忆按人独立，互不干扰。
